server instructions

  • download and install nodejs

  • download and install .net 5 runtime 64-bit

  • open command line and type "node", there should be an output with node version number

  • open command line and type "dotnet", there should be an output with dotnet version number

  • in command line type- cd "c:\serverfolderPath"

  • in command line type- npm i

  • in command line type- cd "c:\serverfolderPath\enableRSAKey"

  • in command line type- dotnet build

  • in command line type- cd "c:\serverfolderPath\enableSSH"

  • in command line type- dotnet build

  • in folder c:\serverfolderPath\enableSSH\bin\Debug\net5.0 make sure chromedriver version matches server chrome version

  • in serverfolderPath open config.json

    • //input true if want to use hostname instead of localhost

    "hostname":false,

    //use enablessh.exe path with double backslashes

    "enableSSHPath":"C:\\ESXI_VCenterClone-Server\\enableSSH\\bin\\Debug\\net5.0",

    //use enableRSAKey.exe path with double backslashes

    "enableRSAPath":"C:\\ESXI_VCenterClone-Server\\enableRSAKey\\bin\\Debug\\net5.0",

    //use putty.exe path with double backslashes

    "puttyPath":"C:\\ESXI_VCenterClone-Server",

    //use plink.exe path with double backslashes

    "plinkPath":"C:\\ESXI_VCenterClone-Server"

  • in command line type- in serverFolderPath root "node app.js"

client instructions

  • in command line type- cd "c:\clientfolderPath"

  • in command line type- npm i

  • to run front-end application type in command line- "npm start"

Operation instructions

  • in order to add new hosts input in the new host form the ip/hostname of the esxi desired

  • if no ssh is enabled use the "enableSSH" button and input the desired data in order to add it

  • to delete hosts from the list of nodes simply drag the desired host to the trash bin

  • to see current power status of each VM click on the host arrow button
